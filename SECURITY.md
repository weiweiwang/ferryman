# Security Policy

## Supported Scope

This repository covers the Ferryman desktop application, bundled Skills,
build scripts, local hooks, and related infrastructure files that are stored
in this source tree.

## Reporting a Vulnerability

Please do not open a public GitHub issue for a suspected security problem.

Instead, report it privately to the maintainer or owning organization with:

- a short summary of the issue
- affected files, features, or release builds
- reproduction steps or a proof of concept
- impact assessment if known
- any suggested remediation

If the issue involves leaked credentials, signing assets, or model provider
tokens, rotate or revoke the secret immediately in the upstream provider.

## Response Expectations

The project should acknowledge valid reports, assess impact, and coordinate a
fix before public disclosure when practical. Please avoid publishing exploit
details until maintainers have had a reasonable chance to respond.

## Out of Scope

- Support questions
- Feature requests
- Issues in third-party model providers or external websites not caused by Ferryman
